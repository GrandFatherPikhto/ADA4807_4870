# Автоматизированный расчёт и симуляция усилителя на ADA4870 с предусилителями ADA4807-2

Данный проект автоматизирует подбор номиналов резисторов и конденсатора обратной связи для выходного каскада на ADA4870, управляемого от дифференциальных токовых выходов ЦАП (моделируется поведенческими источниками тока). Производится расчёт параметров, автоматическая подстановка в схему LTspice, запуск симуляции, анализ THD, построение графиков и генерация отчётов.

## Возможности

- Расчёт номиналов `Ra`, `Rf` (резисторы обратной связи ADA4870), `Rb` (балансировочный резистор) и `Cf` (корректирующая ёмкость) на основе требований к выходной амплитуде, току полной шкалы ЦАП, сопротивлению нагрузки и напряжению питания.
- Выбор ближайших стандартных значений из ряда E96 (1%).
- Сортировка комбинаций по ошибке балансировки `Rb` и близости `Rf` к рекомендованному значению (1210 Ом для CFB-усилителя).
- Автоматическая запись номиналов в схему LTspice (файл `.asc`).
- Запуск переходного процесса (`.tran`) с адаптивным временем, обеспечивающим целое число периодов для Фурье-анализа.
- Сохранение всех сигналов в CSV.
- Построение графиков: выходное напряжение, ток нагрузки, спектр гармоник, входные токи ЦАП (рассчитываются по напряжениям на резисторах TIA), деградация THD от частоты.
- Генерация netlist’а исходной схемы с очисткой от комментариев и созданием читаемого отчёта о связях (файлы `.clean.net.txt` и `_readable.txt`).
- Подробный текстовый отчёт (`report.txt`) со всеми использованными параметрами и результатами симуляции.

## Установка и зависимости

1. **Python 3.10+**.
2. Установите необходимые библиотеки:

```bash
pip install PyLTSpice matplotlib pandas numpy
```

3. Убедитесь, что LTspice установлен и путь к исполняемому файлу правильно указан в `config.json` (ключ `ltspice.executable`).
4. Файл схемы `ada4807_4870.asc` должен лежать в той же папке, что и `main.py`. В схеме должны присутствовать компоненты с именами, используемыми скриптом:
   - Резисторы: `Ra`, `Rb`, `Rf`, `Rfn`, `Rfp`, `Rload`
   - Конденсатор: `Cf`
   - Источники тока: `B§IOUTA`, `B§IOUTB` (или `BIOUTA`, `BIOUTB` в .asc)
   - Поведенческие параметры: `IFS`, `FREQ` (могут быть заданы как `.param`)

## Конфигурация (`config.json`)

Пример конфигурационного файла с пояснениями:

```json
{
    "schematic": {
        "path": "ada4807_4870.asc"
    },
    "ltspice": {
        "executable": "C:\\Users\\grand\\AppData\\Local\\Programs\\ADI\\LTspice\\LTspice.exe"
    },
    "simulation": {
        "output_dir": "./out",
        "temp_dir": "./temp"
    },
    "params": {
        "I_FS": 0.004,               // ток полной шкалы ЦАП, А
        "R_TIA": 249,                // резисторы обратной связи TIA (Rfn, Rfp), Ом
        "V_out_amp": 14.0,           // желаемая амплитуда на нагрузке, В
        "R_load": 22,                // сопротивление нагрузки, Ом
        "V_sup": 18,                 // напряжение одной шины питания (положительной), В
        "V_headroom": 2.0,           // запас до клиппирования (от каждой шины), В
        "I_out_max": 1.0,            // максимальный выходной ток ADA4870, А
        "Rf_max": 10000,             // максимально допустимый Rf, Ом
        "Ra_candidates": [50, 75, 100, 120, 150, 200],  // перебираемые значения Ra
        "C_in_parasitic": 27e-12,    // паразитная ёмкость на входе ADA4870 (Cpara+Ca), Ф
        "Rf_target": 1210            // рекомендованное производителем значение Rf, Ом
    },
    "frequencies": [100000, 500000, 1000000, 2000000, 3000000, 5000000],
    "tran_settings": {
        "periods_transient": 10,     // число периодов на переходные процессы
        "periods_analysis": 10,      // число периодов, сохраняемых для анализа
        "points_per_period": 1000    // количество точек на период
    },
    "plots": {
        "time_domain": true,         // график V(signal) и I(Rload)
        "spectrum": true,            // спектр гармоник (столбцы)
        "degradation": true,         // зависимость THD от частоты
        "input_currents": true       // графики токов IOUTA, IOUTB (рассчитанные)
    },
    "netlist_generator": {
        "generate_netlist": true,    // генерировать ли netlist
        "output_dir": "./out/netlist"
    }
}
```

### Примечания по параметрам

- `V_out_amp` – желаемая амплитуда на нагрузке. Если она превышает `V_sup - V_headroom`, будет автоматически уменьшена до безопасного значения.
- `Ra_candidates` – список значений Ra для перебора. Для каждого вычисляется `Rf = A_v * Ra`, округляется до E96, проверяется `Rf_max`.
- `C_in_parasitic` – суммарная ёмкость на инвертирующем входе ADA4870 (в вашей схеме: `Cpara` 5 пФ + `Ca` 22 пФ = 27 пФ).
- `Rf_target` – используется при сортировке комбинаций (чем ближе `Rf_e96` к этому значению, тем лучше).
- `tran_settings` – управляют длительностью симуляции: `periods_transient` периодов на установление, затем `periods_analysis` периодов, которые попадут в CSV и Фурье-анализ.
- `plots` – флаги включения/выключения соответствующих графиков.
- `netlist_generator` – если `true`, перед симуляцией генерируются чистый netlist и читаемый отчёт в папку `output_dir/netlist`.

## Запуск

1. Убедитесь, что все файлы проекта лежат в одной папке, а `config.json` настроен.
2. Запустите скрипт:

```bash
python main.py
```

3. В консоли будет отображаться прогресс: расчёт комбинаций, запуск LTspice, извлечение данных, построение графиков.
4. Результаты сохраняются в папку `out`:
   - `ada4870_raw_export.csv` – все сигналы по времени.
   - `report.txt` – сводка параметров, выбранных номиналов, THD.
   - Если включено – графики отображаются на экране.
   - Если включена генерация netlist – в `out/netlist` будут лежать текстовые описания схемы.

## Логирование

Все действия записываются в файл `simulation.log` (уровень DEBUG) и выводятся в консоль (INFO). Это помогает отследить ошибки (например, если LTspice не найден или схема не содержит нужных компонентов).

## Примечания по схеме LTspice

- Поведенческие источники тока должны иметь имена, начинающиеся с `B` (например, `BIOUTA`, `B§IOUTA`).
- Резисторы обратной связи первого каскада должны называться `Rfn` и `Rfp`.
- Резисторы `Ra`, `Rb`, `Rf`, `Rload`, конденсатор `Cf` – именно так, как указано в `config.json` (регистр важен, но в LTspice обычно нечувствителен).
- Узел выхода должен называться `SIGNAL`.
